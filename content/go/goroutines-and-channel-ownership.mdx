    go func() {
        // The sending goroutine owns closing the output channel.
        defer close(results)

        for {
            select {
            case <-ctx.Done():
                // Context cancellation gives the goroutine a clean stop.
                return
            case job, ok := <-jobs:
                if !ok {
                    return
                }
                results <- process(job)
            }
        }
    }()
}`,

func StopWorker(results chan Result) {
    // Closing a channel from the outside can race with senders.
    close(results)
}`,

## What to review

The good version shows who reads, who writes, who closes the output channel, and how the goroutine exits when the context is canceled.

The bad version leaves lifetime implicit and lets a separate function close a channel it does not own.

## Review notes

In Go concurrency review, ask: who starts this goroutine, who stops it, who closes each channel, and what happens if the receiver leaves early?
