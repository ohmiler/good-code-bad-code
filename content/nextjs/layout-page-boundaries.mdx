## What to review

The good version uses the layout for persistent navigation and places route-specific content in the child page.

The bad version fetches leaf-page data inside the layout, so every child route can be blocked by data it may not need.

## Review notes

In review, ask whether a layout dependency is truly shared by all child routes. The bad version is review-worthy because layouts persist across navigation, so putting page-specific work there can make unrelated pages slower and harder to stream.
