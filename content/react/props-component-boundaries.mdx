## What to review

The good version makes the component boundary obvious: the component receives the display values it needs and renders them.

The bad version reaches into broad application state, which couples a reusable summary to unrelated project, review, and viewer concerns.

## Review notes

In review, ask whether a component can be understood, tested, and reused from its props alone. The bad version is review-worthy because it renders correctly in one app context while hiding dependencies that make the component harder to move or reason about.
