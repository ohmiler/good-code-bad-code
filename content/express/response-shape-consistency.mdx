## What to review

The good version gives routes small helpers for success and error responses, making the API contract consistent.

The bad version returns arrays, objects, strings, and nested data shapes depending on the route.

## Review notes

In review, compare response bodies across neighboring endpoints. The bad version is review-worthy because inconsistent API envelopes push accidental complexity onto every client and test.
