## What to review

The good version exports an app factory, so tests and workers can create the same app without starting a server.

The bad version starts listening as a module side effect. Importing the app for tests or scripts now opens a port unexpectedly.

## Review notes

In review, check whether the Express app can be constructed independently from `listen()`. The bad version is review-worthy because startup side effects make testing, shutdown, and multi-process usage harder than they need to be.
