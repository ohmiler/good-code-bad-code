## What to review

The good version puts the approval rule next to the transformation, so the output list follows from the pipeline.

The bad version builds a mutable list and forgets the score check, making the loop mechanics louder than the domain rule.

## Review notes

Review collection code by naming the rule first: which items stay, which fields leave, and what happens to invalid data. Kotlin pipelines should make those steps visible without hiding expensive work in callbacks.
