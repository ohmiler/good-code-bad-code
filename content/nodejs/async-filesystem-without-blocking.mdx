## What to review

The good version awaits an async file read and lets the event loop continue handling other work while the disk is busy.

The bad version uses `readFileSync` in a request path, blocking every other request in the process until the read finishes.

## Review notes

In review, synchronous filesystem APIs are usually acceptable during startup or build tooling, but suspicious in request handlers. The bad version is review-worthy because it can pass local testing while causing latency spikes under concurrent traffic.
