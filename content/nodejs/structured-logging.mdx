## What to review

The good version emits structured JSON with stable fields that log systems can search and aggregate.

The bad version logs a free-form string and includes a password, turning logs into a security incident.

## Review notes

In review, look for correlation IDs, event names, levels, and sensitive fields. The bad version is review-worthy because logs are production data and should be useful without becoming a place where secrets are copied.
