## What to review

The good version separates transport failures from unsuccessful HTTP responses and only parses JSON after the response is acceptable.

The bad version is concise, but it assumes every request succeeds and every response body can be parsed as JSON.

## Review notes

In review, ask what the caller receives for offline users, server errors, and non-JSON error pages. The bad version is review-worthy because it works on the happy path while pushing important failure behavior into unhandled exceptions.
